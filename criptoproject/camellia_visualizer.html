💡 Mengapa Camellia Cipher?

Dalam sistem SecureMedis, data medis sensitif pasien dilindungi secara End-to-End. Artinya, data dienkripsi sebelum dikirim dan disimpan ke cloud database Supabase.

Flow Pengamanan:
  1. Dokter mengetik rekam medis (Plaintext).
  2. Aplikasi melakukan Camellia Encryption secara lokal.
  3. Hasil enkripsi (Ciphertext) dikirim ke Supabase.
  4. Pasien login & mengunduh data terenkripsi.
  5. Aplikasi melakukan Camellia Decryption secara lokal di HP pasien.

Jika admin Supabase mencoba melihat database secara langsung, mereka hanya akan melihat string acak seperti di atas, bukan rekam medis medis aslinya.
