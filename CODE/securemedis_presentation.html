⚠️ Latar Belakang & Masalah

Kerentanan Privasi Data Medis Pasien

Data rekam medis (Diagnosis, Resep, Catatan Dokter) bersifat sangat sensitif dan dilindungi undang-undang. Namun, pada arsitektur cloud database tradisional:

  • Kebocoran Cloud: Jika server cloud ditembus hacker, semua data rekam medis pasien akan langsung terekspos plaintext.
  • Adanya Orang Dalam: Administrator database/server cloud memiliki akses penuh untuk membaca data riwayat medis pasien.
  • Kredensial Lemah: Password yang disimpan tanpa hashing yang kuat dapat diretas secara brute-force.

🛡️ Solusi Sistem Keamanan

End-to-End Encryption & Hak Akses Berbasis Role

🩺

DOKTER (Akses Tulis & Baca)

Dokter menginput rekam medis. Sebelum data dikirim ke internet, perangkat dokter mengenkripsinya dengan **Camellia-128**. Dokter hanya bisa melihat data pasien miliknya sendiri.

👤

PASIEN (Akses Baca Saja)

Pasien mengunduh data miliknya dalam keadaan terenkripsi dari Supabase. Aplikasi mendekripsinya secara lokal di memori HP pasien. Pasien tidak bisa memodifikasi data atau melihat rekam medis pasien lain.

⚙️

ADMIN (Akses Manajemen)

Admin hanya bertugas mendaftarkan/menghapus user (Dokter/Pasien). Admin **sama sekali tidak memiliki akses read maupun write** ke tabel rekam medis. Data aman dari penyalahgunaan admin.

🔑 BCrypt Hashing

Kredensial login diamankan dengan **BCrypt** di sisi database. Password tidak pernah disimpan dalam bentuk plaintext. Saat login, aplikasi memverifikasi string password secara aman terhadap hash BCrypt cost=12 yang disimpan di database Supabase.

🧮 Spesifikasi Cipher

Mengenal Algoritma Camellia

Camellia adalah block cipher simetris 128-bit yang dikembangkan bersama oleh **Mitsubishi Electric** dan **NTT Jepang** pada tahun 2000. Algoritma ini memiliki efisiensi dan tingkat keamanan yang setara dengan AES (Advanced Encryption Standard).

Ukuran Blok 128-bit (16 Byte)
Ukuran Kunci 128, 192, atau 256 bit
Struktur Putaran 18 Rounds Feistel
Fungsi Linear FL & FL-inv setiap 6 round

Struktur Inti Feistel

Feistel Network membelah blok data 128-bit menjadi dua bagian 64-bit: Left (L) dan Right (R).

L_baru = R_lama
R_baru = L_lama XOR F(R_lama, Subkey_i)

F-Function terdiri dari:
1. XOR data 64-bit dengan subkey.
2. Substitusi byte melalui 4 buah S-Box.
3. Difusi bit melalui P-Function (matriks MDS).
